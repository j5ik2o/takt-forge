# Product Overview

`qfind` は、開発者向けの高速なファイル検索 CLI です。  
ファイル名（glob）とファイル内容（regex）の条件を組み合わせ、コードベース内の候補を絞り込む用途に特化しています。

## Core Capabilities

- ディレクトリ配下を並列走査し、候補ファイルを高速に収集する
- `--glob` でファイル名条件を指定して候補を絞り込む
- `--regex` でファイル内容条件を指定してさらに絞り込む
- `.gitignore` を尊重して不要なファイルを探索対象から除外する
- プレーンテキスト出力と JSON 出力（`--json`）を切り替える
- `--max-depth` で探索の深さを制限する

## Target Use Cases

- 大規模リポジトリで、条件付きのファイル探索を高速に行いたい
- スクリプトや他ツールから、機械可読な検索結果（JSON）を受け取りたい
- `.gitignore` を前提に、実作業対象のみを検索したい
- 深い階層を全探索せず、調査範囲を制御したい

## Value Proposition

- `find` と `grep` の組み合わせで手作業していた探索を、単一コマンドで完結できる
- 並列走査と決定論的な結果順序を両立し、速度と再現性を確保している
- 入力条件を型（`FileNamePattern` / `ContentPattern` / `DepthLimit`）として早期に確定し、実行時の不正状態を減らしている
- エラーを CLI 利用者向けに整理し、終了コードとメッセージを一貫して返す
